# Results

The project delivered a complete pipeline from data acquisition to risk monitoring. Key results include:

- **Clean dataset**: A harmonised and cleaned transaction dataset stored in `data/processed/`, ready for modelling.

- **Insightful EDA**: Visualisations revealing patterns in transaction amounts, typologies and temporal behaviour.

- **High‑performing model**: The XGBoost classifier achieved a recall of ~0.92 and an AUC of 0.98, outperforming baseline models. It demonstrated the ability to detect the majority of suspicious transactions with manageable false positives.

- **Interactive dashboard**: The Power BI report integrates model outputs with visualisations, enabling analysts to explore patterns and investigate flagged cases.

- **Reproducible notebooks**: Each step of the workflow is documented in Jupyter notebooks to facilitate transparency and reproducibility.

These results show that data‑driven approaches can significantly enhance transaction monitoring and support AML compliance efforts.
