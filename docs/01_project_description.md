# Project Description and Objectives

This graduation project explores **transaction detection and risk monitoring** in financial services.  
The objective is to design a data‑driven system that can help compliance teams identify suspicious transactions and monitor risk patterns in a large-scale payments dataset.  
To accomplish this goal we built an end‑to‑end analytics pipeline that includes data ingestion, exploration, modelling and dashboarding.  

The key objectives are:

1. **Extract, transform and load (ETL) the raw transaction data** into a clean format suitable for analysis.  
2. **Understand the statistical properties of the data** through exploratory data analysis (EDA) and summarise the distribution of normal and suspicious behaviour.  
3. **Develop machine‑learning models** that can classify transactions as “normal” or “suspicious” based on historical patterns.  
4. **Build interactive dashboards** that allow stakeholders to visualise risk metrics, monitor transaction flows and inspect flagged cases.  
5. **Evaluate business impact** by measuring model performance and demonstrating how early detection improves compliance.

The remainder of the documentation explains how we acquired the data, processed and analysed it, built models, and deployed the results.
